# Инструмент Bitrix/Aspro: массовая деактивация и удаление товаров по CODE из XLSX

## Назначение
Этот инструмент добавляет в админку Битрикс страницу для безопасной пакетной обработки товаров по символьному коду (`CODE`) из XLSX-файла.

Возможности:
- загрузка `.xlsx` с кодами из первой колонки `A`;
- поиск элементов в выбранном инфоблоке (`IBLOCK_ID`) по точному совпадению `=CODE`;
- режимы работы:
  - **Только отчет (dry-run)**;
  - **Снять с публикации (ACTIVE = N)**;
  - **Удалить** (только если элемент уже неактивен).

## Важные предупреждения
1. Перед массовыми действиями обязательно сделайте **резервную копию** сайта и БД.
2. Рекомендуемый безопасный сценарий: сначала запуск в режиме **dry-run**, затем **деактивация**, и только после проверки — **удаление**.
3. Если по одному `CODE` найдено несколько элементов (дубликат), инструмент **не изменяет** такие записи автоматически и выводит их в отчет.
4. Для режимов изменения данных (деактивация/удаление) необходимо включить чекбокс: **«Я понимаю последствия»**.

## Установка

### 1) Разместите файлы
Скопируйте файлы в проект:

- `/bitrix/admin/aspro_delete_by_codes.php` (обертка)
- `/local/admin/aspro_delete_by_codes.php` (основная логика)

### 2) Установите PhpSpreadsheet
Инструмент использует библиотеку `phpoffice/phpspreadsheet` через Composer-автозагрузчик `/local/vendor/autoload.php`.

Выполните в корне проекта:

```bash
composer require phpoffice/phpspreadsheet
```

Если Composer в окружении не настроен — установите его стандартным для вашего сервера способом.

### 3) Проверьте права доступа
- Доступ к инструменту должен быть только у администраторов или пользователей с правами записи в целевой инфоблок.
- Убедитесь, что веб-сервер может создавать/записывать:
  - `/upload/tmp/` (временный XLSX);
  - `/bitrix/logs/aspro_delete_by_codes.log` (файл лога).

## Как открыть инструмент в админке
Откройте URL:

```text
/bitrix/admin/aspro_delete_by_codes.php
```

## Как пользоваться
1. Укажите **IBLOCK_ID** каталога (обязательно).
2. Загрузите `.xlsx` файл.
3. Выберите режим:
   - **Только отчет (dry-run)** — только проверка и отчет;
   - **Снять с публикации** — выставляет `ACTIVE = N` для найденных уникальных совпадений;
   - **Удалить** — удаляет только элементы с `ACTIVE = N`.
4. Для режимов изменения данных установите чекбокс **«Я понимаю последствия»**.
5. Нажмите **«Запустить»**.
6. Проверьте сводку и детальный отчет.

## Ограничения
- Читается **только первая колонка `A`**.
- Обрабатывается только поле элемента инфоблока **`CODE`**.
- Пустые строки игнорируются.
- Коды в файле приводятся к `trim` и дедуплицируются (`unique`).
- Есть лимит на количество уникальных кодов за запуск: **5000**.
- Если в инфоблоке по одному коду найдено больше одного элемента (дубликаты), инструмент не деактивирует и не удаляет их автоматически.

## Логирование
Лог пишется в файл:

```text
/bitrix/logs/aspro_delete_by_codes.log
```

Формат логов включает дату/время, `user_id`, режим, `iblock_id`, количество кодов и итог результата выполнения.
